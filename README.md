        # redis — Ключи и TTL

        Homework-шаблон для урока **l3_keys_and_ttl** (Ключи и TTL) на платформе Vibe Learn.

        ## Что делать

        Напиши expiration-monitor на go-redis, который ловит истечения TTL через keyspace notifications:
- в начале включи нотификации: `CONFIG SET notify-keyspace-events Ex` (E = keyevent, x = expired);
- подпишись на канал `__keyevent@0__:expired` (0 — номер БД) через PSubscribe/Subscribe;
- на каждое событие истечения логируй structured-запись: `{ts, key, db}` в JSON;
- корректно завершайся по SIGINT (graceful shutdown: отписка + закрытие соединения).
Тесты создадут N ключей с короткими TTL (`SET k v PX 200`), подождут истечения и проверят:
(1) monitor поймал ровно те ключи, что истекли; (2) события пришли с правильным db-индексом;
(3) shutdown по сигналу не теряет уже полученные события и не паникует.
Важный нюанс, который проверяет один из тестов: notify-keyspace-events НЕ доставит событие,
если на канал никто не подписан в момент истечения — это best-effort, не durable-очередь.

## Контекст (из transfer-задачи урока)

Веб-приложение, 100k активных юзеров, средний размер сессии ~2KB. Сессии хранятся в Redis
с TTL=24 часа. Memory limit на Redis = 4GB. После запуска фичи «remember me» на 30 дней
память начала расти линейно — за неделю +1.5GB, ещё через неделю Redis ляжет с OOM.
Что не так и какие три варианта решения у тебя есть (с trade-off'ами)?

## Recap из урока

- Имена ключей: `:` как разделитель, тип объекта первым (`user:42:profile`). Хорошие имена → проще SCAN по префиксу + Cluster hash-теги.
- TTL ставится через `EX/PX` опцию SET или отдельной `EXPIRE`. Без TTL ключ живёт до DEL или maxmemory-вытеснения.
- Просрочка обрабатывается lazy (при чтении) + active (раз в 100мс сэмплируется 20 случайных ключей). Память может временно превышать «живой» размер.
- `allkeys-lfu` — обычно лучшая политика для кэш-сервера: LFU устойчив к всплескам, LRU нет.
- Много мелких объектов с одинаковыми полями → переложи в хеш. Экономия памяти 3-5× за счёт ziplist-encoding.

        ## Как работать

        1. Платформа Vibe Learn создаёт копию этого репо в твоём GitHub-аккаунте по клику «Начать домашку» на странице урока (через GitHub `/generate`, codecrafters-pattern).
        2. Склонируй копию локально, реализуй TODO в `main.go`, прогони тесты, запушь.
        3. CI (`.github/workflows/ci.yml`) запускает `go vet` + `go test ./...` на каждый push. Платформа слушает результат через webhook от GitHub Actions и обновляет статус домашки на странице урока.

        ## Локальное окружение

        - Go 1.22+
        - Docker + docker-compose — `docker compose up -d` поднимает single-node Redis 7 на `localhost:6379` (с включёнными keyspace-notifications и AOF). Адрес переопределяется через env `REDIS_ADDR`.

        ## Запуск

        ```bash
        # Поднять локальный Redis
        docker compose up -d

        # Прогнать тесты (интеграционный включается через REDIS_INTEGRATION=1)
        go test ./...
        REDIS_INTEGRATION=1 go test ./...

        # Запустить main (печатает marker; замени stub на реализацию)
        go run .
        ```
